GROM Web

Checklist de Hospedagem

Itens que precisam estar prontos antes de colocar o sistema no ar.

Foco: estabilidade, acesso, impressao e operacao
Estado dos cadastros faltantes: podem ficar para depois da hospedagem
A regra de ouro dos relatorios e timbrados ja esta consolidada. Antes da hospedagem, o mais importante e garantir ambiente, seguranca, build e validacao final.

1. Infraestrutura

  • Servidor ou VPS com PHP 8.3 ou superior.
  • Banco de dados de producao definido e acessivel.
  • Redis habilitado, se a fila, cache e sessao continuarem nele.
  • Dominio ou subdominio apontando para a pasta public/.
  • HTTPS ativo com certificado valido.

2. Ambiente (.env)

  • APP_ENV=production.
  • APP_DEBUG=false.
  • APP_URL correto para o endereco publico.
  • DB_* apontando para a base real.
  • MAIL_* configurado se houver envio de e-mail.
  • GROM_REQUIRE_PASSWORD_CHANGE=true no go-live.
  • GROM_BOOTSTRAP_ADMIN_* preenchido para o admin inicial.

3. Deploy

  • composer install --no-dev.
  • npm install.
  • npm run build.
  • php artisan key:generate se a chave ainda nao existir.
  • php artisan migrate --force.
  • php artisan storage:link.

4. Cache e Performance

  • php artisan config:cache.
  • php artisan route:cache.
  • php artisan view:cache.
  • Autoload otimizado e dependencias de producao limpas.
  • Arquivos de storage e cache com permissao de escrita.

5. Operacao Continua

  • Worker de fila configurado e monitorado.
  • Scheduler do Laravel ativo no servidor.
  • Backup automatico da base antes do uso real.
  • Log e rotacao de logs conferidos.
  • Espaco em disco suficiente para PDF, relatarios e anexos.

6. Validacao Pos-Subida

  • Login e acesso por perfil.
  • Modulo RH.
  • Escala mensal e plantoes externos.
  • Mandados e relatorios PDF.
  • Timbrado consolidado em todas as impressoes.
  • Impressao sem sobrepor cabecalho ou rodape.

7. Pendencias Aceitas Depois da Hospedagem

  • Cadastro manual das funcionarias faltantes no RH: Lara, Larissa e Emanuelle.
  • Ajustes finos de diagrama somente se surgir algum caso real em producao.
  • Eventual revisao dos acessos por perfil conforme o uso do dia a dia.

8. Decisao Sobre o Legado

  • Definir se a sincronizacao legada continua ativa em producao.
  • Se continuar, garantir acesso ao arquivo legado usado pelo sincronizador.
  • Se nao continuar, trabalhar apenas com os cadastros do PHP.
