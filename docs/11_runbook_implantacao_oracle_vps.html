GROM Web

Runbook de Implantacao Oracle VPS

Objetivo: reduzir a hospedagem a preencher o arquivo de ambiente, executar scripts prontos e validar um conjunto curto de checks.

Dominio: grom.seg.br
Stack: Nginx + Docker + PostgreSQL + Redis
Modo: baixa interferencia humana
Arquivos que ja existem no repositorio
  • infra/.env.production.example
  • infra/docker-compose.prod.yml
  • infra/nginx/grom.seg.br.conf.example
  • infra/scripts/deploy-prod.sh
  • infra/scripts/healthcheck-prod.sh
  • infra/scripts/backup-prod.sh

1. Preparar a VPS

sudo apt update sudo apt install -y ca-certificates curl gnupg nginx certbot python3-certbot-nginx git ufw curl -fsSL https://get.docker.com | sudo sh sudo usermod -aG docker $USER newgrp docker sudo ufw allow OpenSSH sudo ufw allow 'Nginx Full' sudo ufw enable
  • Docker responde com versao valida
  • Nginx instalado no host
  • DNS de grom.seg.br apontado para a VPS

2. Publicar o codigo

sudo mkdir -p /opt/grom sudo chown $USER:$USER /opt/grom git clone <REPOSITORIO> /opt/grom/grom_web_php cd /opt/grom/grom_web_php/infra cp .env.production.example .env.production

Preencher somente estes campos obrigatorios em .env.production:

  • APP_KEY
  • DB_PASSWORD
  • GROM_BOOTSTRAP_ADMIN_PASSWORD
  • MAIL_FROM_ADDRESS, se necessario
docker run --rm -v /opt/grom/grom_web_php/runtime:/app -w /app php:8.4-cli-alpine php artisan key:generate --show

3. Subir a aplicacao

cd /opt/grom/grom_web_php/infra bash ./scripts/deploy-prod.sh
  • stack sobe sem erro
  • migrations aplicadas
  • seed estrutural aplicada
  • login interno responde

4. Configurar HTTPS

sudo cp /opt/grom/grom_web_php/infra/nginx/grom.seg.br.conf.example /etc/nginx/sites-available/grom.seg.br.conf sudo ln -s /etc/nginx/sites-available/grom.seg.br.conf /etc/nginx/sites-enabled/grom.seg.br.conf sudo nginx -t sudo systemctl reload nginx sudo certbot --nginx -d grom.seg.br sudo nginx -t sudo systemctl reload nginx

5. Validar tudo

cd /opt/grom/grom_web_php/infra bash ./scripts/healthcheck-prod.sh

Depois, rodar uma carga curta a partir de uma maquina autorizada:

powershell -ExecutionPolicy Bypass -File .\scripts\load-test.ps1 -BaseUrl https://grom.seg.br -FailOnThreshold
  • login abre em HTTPS
  • dashboard abre sem erro 500
  • RH e escalas abrem
  • PDF responde
  • sem servico unhealthy

6. Backup operacional

cd /opt/grom/grom_web_php/infra bash ./scripts/backup-prod.sh

O backup SQL sera gravado em /opt/grom/backups por padrao.

7. Regra de corte

  • APP_DEBUG continua false
  • seed demo continua desabilitada
  • sync legado continua desabilitado
  • backup validado
  • carga curta sem HTTP 500

8. Plano B imediato

Se a publicacao falhar, usar o roteiro de rollback ja preparado em docs/10_rollback_oracle_vps.md.

Documento de mesa para impressao. Fluxo minimo: preencher .env.production, executar deploy-prod.sh, configurar HTTPS, executar healthcheck-prod.sh, rodar backup e validar carga curta.
